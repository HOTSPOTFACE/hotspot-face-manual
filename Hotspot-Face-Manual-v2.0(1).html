1. Introduction

The Hotspot-Face is an advanced touchscreen monitor specifically designed for DMR hotspot systems operating PI-Star. It provides real-time visual feedback of radio transmissions, displaying callsigns, talk groups, and transmission durations in an intuitive, color-coded interface.

This device eliminates the need to constantly check your computer or mobile device to monitor Pi-Star activity, providing an always-on, dedicated display that sits conveniently on your desk or in your radio shack.

Key Benefits:

  • Instant visual feedback of DMR activity
  • No computer required for monitoring
  • Touch-enabled interface for easy configuration
  • Automatic network detection and configuration
  • Dual-mode operation (WiFi and Hotspot)

3. Initial Setup

First Time Setup

When you power on your Hotspot-Face for the first time, it will automatically enter Access Point mode for initial configuration.

  1. Power On: Connect the Hotspot-Face to a USB power source using the included USB-C cable
  2. Wait for AP Mode: The screen will display "Setup Mode" with connection instructions
  3. Connect to WiFi:
    • On your phone or computer, connect to the WiFi network: Hotspot-FACE Setup
    • Enter the password: M3URE1512
  4. Open Configuration Page:
    • Open a web browser
    • Navigate to: 192.168.4.1
    • The configuration page will open automatically
  5. Configure Settings:
    • Display Name: Enter a custom name for your device
    • Connection Mode: Select your setup type:
      • "WiFi Only" - Pi-Star on your home network
      • "Hotspot Only" - Direct connection to Pi-Star hotspot
      • "Both" - Automatic switching between modes
    • WiFi Settings: Select or enter your WiFi network credentials
    • Pi-Star IPs: Enter the IP addresses for your Pi-Star:
      • WiFi IP: Usually something like 192.168.1.100
      • Hotspot IP: Usually 192.168.50.1 or 10.0.0.1
    • Auto-Discovery: Enable to automatically find Pi-Star on your network
  6. Save Configuration: Click "Save & Connect"
  7. Restart: The device will restart and connect to your network

4. Display Layout

The Hotspot-Face display is organized into distinct color-coded sections for easy reading:

Screen Layout

Header Bar Displays your custom device name. Touch to access settings.
Status Box Shows current status (Idle/TX/Error)
Callsign Box Displays the transmitting station's callsign
Talk Group Box Shows the active talk group or reflector

Date Bar Shows current date and day of week

Status Indicators

  • Idle: No active transmission
  • TX: Active transmission (displayed in red)
  • WiFi Off: No WiFi connection

5. Normal Operation

Automatic Operation

Once configured, the Hotspot-Face operates automatically:

  • Connects to your network on power-up
  • Automatically detects network type (WiFi or Hotspot)
  • Finds and connects to Pi-Star
  • Updates display every second
  • Shows transmission activity in real-time

Network Mode Switching

The Hotspot-Face intelligently switches between network modes:

  • WiFi Mode: When connected to your home network
  • Hotspot Mode: When connected directly to Pi-Star's hotspot
  • Automatically selects the appropriate Pi-Star IP address
  • Connection status shown in bottom bar

Touch Interactions

  • Header Bar (Quick Press): Opens settings menu
  • Header Bar (10-second hold): Factory reset

6. Settings Menu

Accessing Settings

Touch and release the dark blue header bar to open the settings menu.

Settings Options

Setting Description
Display Name Change the text shown in the header bar
WiFi SSID Change your WiFi network
WiFi Password Update WiFi password
Pi-Star WiFi IP IP address when on WiFi network
Pi-Star Hotspot IP IP address when Pi-Star is hotspot
Auto-Discovery Toggle automatic Pi-Star detection
EXIT Return to main display

On-Screen Keyboard

When editing text fields, an on-screen keyboard appears with:

  • Full QWERTY layout
  • Numbers and symbols (press "123" to switch)
  • Shift key for uppercase
  • Delete key for corrections
  • Space bar
  • OK button to save changes

7. Advanced Features

Auto-Discovery (mDNS)

When enabled, the Hotspot-Face automatically searches for Pi-Star on your network using:

  • mDNS/Bonjour protocol
  • Common Pi-Star hostnames
  • Network scanning of likely IP addresses
  • Automatic validation of discovered devices

Dual IP Management

The Hotspot-Face stores separate IP addresses for different connection scenarios:

  • WiFi IP: Used when connected to your home network
  • Hotspot IP: Used when connected to Pi-Star's hotspot
  • Automatic detection of network type
  • Seamless switching between modes

Connection Recovery

The device includes automatic recovery features:

  • Automatic reconnection after network loss
  • Fallback to alternate IP if primary fails
  • Manual reconnection via status bar touch
  • Connection status indicators

Data Persistence

All data is preserved across power cycles:

  • Network settings
  • Pi-Star IP addresses
  • Display preferences
  • Last known good configuration

8. Factory Reset

Warning: Factory reset will erase all settings and return the device to its initial state!

To Perform a Factory Reset:

  1. Press and hold the dark blue header bar
  2. Continue holding for 10 seconds
  3. The display will show a countdown: "Reset in X seconds"
  4. After 10 seconds, "RESETTING..." will appear
  5. Release the header bar
  6. The device will restart in setup mode

After a factory reset, you'll need to reconfigure all settings through the web portal.

9. Troubleshooting

Common Issues and Solutions

Problem Solution
Display shows "WiFi Off"
  • Check WiFi credentials in settings
  • Verify router is powered on
  • Move device closer to router
Display shows "No Pi-Star"
  • Verify Pi-Star IP address is correct
  • Ensure Pi-Star is powered on
  • Check network connectivity
  • Enable Auto-Discovery
Display shows "Auth Error"
  • Default credentials: pi-star / raspberry
  • Update credentials if changed
  • Check Pi-Star admin interface
Touch screen not responding
  • Clean screen with soft cloth
  • Perform factory reset
  • Check for firm touches
Talk Group data disappears
  • Normal when no active transmission
  • Data persists through menu navigation
  • Check Pi-Star dashboard for activity
Cannot access web portal
  • Ensure connected to "Hotspot-Face Setup" WiFi
  • Try 192.168.4.1 in browser
  • Disable mobile data on phone

LED Indicators

  • Power LED (if present): Should be solid when powered
  • Display Backlight: Should be on when active

10. Technical Information

Network Requirements

  • 2.4GHz WiFi network (5GHz not supported)
  • WPA/WPA2 security (WEP not recommended)
  • DHCP enabled on router
  • Access to Pi-Star on same network

Pi-Star Requirements

  • Pi-Star version 3.4 or later
  • Admin interface enabled
  • Network access enabled
  • Valid amateur radio callsign configured

Power Requirements

  • 5V DC via USB-C
  • Minimum 500mA current capacity
  • Typical consumption: 200mA
  • Peak consumption: 350mA

Environmental Specifications

  • Operating Temperature: 0°C to 70°C
  • Storage Temperature: -20°C to 80°C
  • Humidity: 10% to 90% non-condensing
  • Altitude: Up to 3000m
