1. Introduction

The Hotspot-Face is an advanced touchscreen monitor specifically designed for DMR hotspot systems operating PI-Star.

It provides real-time visual feedback of radio transmissions including callsigns, talk groups, connection status, and transmission duration.

Key Benefits

  • Instant DMR activity display
  • No computer needed
  • Touch-based settings
  • Automatic network detection
  • Dual-mode WiFi/Hotspot operation

2. Features

Display

  • 2.8" TFT touchscreen (320×240)
  • Color-coded status boxes
  • Real-time DMR monitoring

Network

  • 2.4GHz WiFi
  • Auto mode detection
  • Auto-discovery for Pi-Star

Configuration

  • On-screen keyboard
  • Captive setup portal
  • Factory reset

3. Initial Setup

On first boot, the device enters AP (Setup) mode.

  1. Power on using USB-C
  2. Connect to WiFi: Hotspot-FACE Setup
    Password: M3URE1512
  3. Open browser → 192.168.4.1
  4. Enter WiFi SSID + password
  5. Enter Pi-Star IPs
  6. Save and reboot
Tip: If you don't know your Pi-Star's IP address, enable Auto-Discovery and the Hotspot-Face will find it automatically.

4. Display Layout

HeaderDevice name + settings access
StatusIdle / TX / Error
CallsignActive station callsign
Talk GroupActive TG/Reflector
DurationTX duration
DateCurrent date
ConnectionWiFi/Hotspot + IP

Status Indicators

  • Idle: No active transmission
  • TX: Active transmission (displayed in red)
  • WiFi Off: No WiFi connection
  • No Pi-Star: Cannot connect to Pi-Star
  • Auth Error: Incorrect Pi-Star credentials
  • Connected successfully
  • Connection failed

5. Normal Operation

  • Auto network detection
  • Automatic Pi-Star connection
  • Live display updates

6. Settings Menu

Tap the header to access settings.

  • Display Name
  • WiFi SSID + Password
  • Pi-Star WiFi & Hotspot IP
  • Auto-Discovery toggle

7. Advanced Features

Auto-Discovery

Uses mDNS + ping scanning to detect Pi-Star on your LAN.

Dual IP Storage

Stores both WiFi and Hotspot IPs for seamless switching.

Connection Recovery

  • Auto reconnect
  • Fallback IP switching

8. Factory Reset

This will erase all stored settings.
  1. Press & hold header for 10 seconds
  2. Wait for countdown
  3. Device reboots into Setup Mode

9. Troubleshooting

No Pi-Star

  • Verify IP
  • Ensure Pi-Star is powered

Auth Error

  • Default login: pi-star / raspberry

No Web Portal

  • Visit 192.168.4.1 in AP mode

10. Technical Information

  • WiFi: 2.4GHz only
  • Power: 5V USB-C
  • Typical: 200mA
  • Operating temp: 0–70°C

11. Safety
